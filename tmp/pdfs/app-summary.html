Repo Summary

Claude4Marketing

A marketing-focused blog site built with Eleventy and managed largely through Claude Code. Repo evidence shows a static content workflow with Markdown posts, Nunjucks layouts, build-time search indexing, and Netlify publishing.

Who It's For

Primary persona: marketers and builders experimenting with AI-assisted publishing, content workflows, and Claude-based marketing tooling.

What It Does

  • Publishes blog posts from Markdown files in src/posts/.
  • Renders a homepage with post cards, dates, tags, and reading-time estimates.
  • Generates individual post pages with related posts, share links, and code highlighting.
  • Provides tag archive pages plus RSS feed, sitemap, robots.txt, and a custom 404 page.
  • Builds client-side search using Pagefind and a dedicated /search/ page.
  • Supports downloadable Markdown assets through passthrough copies to /downloads/.
  • Adds browser-side post reactions backed by Supabase RPC/data calls.

How To Run

1

Install dependencies: npm install

2

Start local dev server: npm start

3

Build production output: npm run build

4

Built site is written to _site/; Netlify is configured to publish that directory.

Not found in repo: automated tests, a separate backend service definition, or a dedicated admin/CMS app.

How It Works

Content
Markdown posts plus metadata in src/posts/ and src/_data/metadata.json.
Build
.eleventy.js wires Eleventy, RSS, date/reading-time/related-post filters, tag collection, and passthrough static assets.
Templates
Nunjucks layouts/pages in src/_includes/ and top-level .njk files render the blog, post, tags, search, feed, sitemap, and 404 routes.
Search
npm run build runs Eleventy first, then Pagefind indexes _site/ for browser search UI.
Runtime
Browser loads static HTML/CSS/JS; post pages also call Supabase directly from the client to read and increment reaction counts.
Deploy
netlify.toml sets npm run build as the build command and _site as the publish directory.
