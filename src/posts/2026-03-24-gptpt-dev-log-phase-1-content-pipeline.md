## Where we are

Phase 1 of GPTPT is a chat-based AI personal trainer — no app, no database, just a system prompt that turns ChatGPT or Claude into a structured coaching session. The product itself is simple by design. The infrastructure behind getting it in front of people is where the engineering starts.

This week I built the first end-to-end content pipeline: from a Notion content brief to rendered Instagram carousel slides, with captions generated by Claude and posts queued for publishing.

## What's automated

### 1. Notion as the source of truth

Every post starts as a page in a Notion database (GPTPT Content Pipeline). The page holds the topic, slide copy, image notes, tone guidance, and post date. The pipeline reads directly from Notion via MCP — no copy-pasting, no manual export.

When a post is ready to go, the status updates automatically from Idea → Ready and the post date is written back to the page.

### 2. Slide rendering via Playwright

Each carousel slide is defined as HTML with inline styles — dark background, amber accent, custom typography — and rendered to a 1080×1080 PNG using Playwright's headless Chromium. Real photos (gym, couch scene) are embedded as base64 and composited directly into the layout.

Five slides per post, each with a specific layout role: hook, content, content+photo, roadmap, CTA. The render script runs in under 10 seconds.

### 3. Caption generation via Claude API

Three caption variants are generated per post — result hook, problem hook, and punchy — using a structured prompt sent to `claude-sonnet-4-6`. Each variant includes a full Instagram caption and hashtag set. The chosen variant is saved to a text file alongside the rendered PNGs.

### 4. Buffer push (in progress)

The final step pushes the rendered images and caption to Buffer for scheduling. The image upload and post creation flow is scripted — currently working through an API auth issue on the Buffer side.

## Stack

- **Notion MCP** — content brief, status updates, post date writes
- **Playwright** — headless HTML-to-PNG rendering
- **Claude API** — caption generation (3 variants per post)
- **Buffer API** — scheduling (in progress)
- **Node.js** — orchestration throughout

## What's next

Buffer auth sorted, then the full pipeline runs in a single command: read brief from Notion → render slides → generate caption → push to Buffer → update Notion status. Zero manual steps between brief and scheduled post.

Phase 2 adds memory — a Notion database that logs every training session the AI coaches, so progress tracking becomes persistent rather than per-conversation.
